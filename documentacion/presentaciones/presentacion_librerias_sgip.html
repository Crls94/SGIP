Objetivo del proyecto SGIP

SGIP automatiza procesos críticos de Metro Ica: inventario, pedidos, alertas, reportes y predicción de demanda.

Objetivo general

Implementar un sistema web integral para controlar stock, gestionar pedidos y apoyar decisiones gerenciales con datos.

Problemas que resuelve

Diferencias entre stock físico y digital, desorden en pedidos local/delivery y ausencia de predicción de demanda.

Solución tecnológica

API REST en Spring Boot, frontend React, PostgreSQL centralizado y módulo IA en Python.

Manejo de archivos y generación de reportes

SGIP necesita reportes exportables para inventario y pedidos, alineados con el requerimiento RF-09.

Apache POI

Genera archivos Excel .xlsx desde Java. Se elige porque permite crear hojas, filas y celdas para reportes administrativos analizables.

Apache PDFBox

Genera documentos PDF desde Java. Se elige porque produce reportes formales, compartibles y con formato fijo.

Conclusión

Las librerías elegidas permiten construir SGIP de forma segura, mantenible y alineada con sus requerimientos funcionales.

Seguridad

Spring Security, BCrypt y JJWT protegen usuarios, roles y endpoints críticos.

Persistencia

Spring Data JPA y PostgreSQL aseguran datos centralizados y transacciones confiables.

Reportes

Apache POI y PDFBox permiten Excel/PDF para operación y gerencia.

Interfaz

React, Axios, Router y Recharts ofrecen una SPA modular y visual.

IA

Pandas y scikit-learn transforman datos históricos en predicciones semanales.

Evolución

OpenAPI, Flyway, Actuator y Testcontainers fortalecen producción y mantenimiento.

Las librerías no solo aceleran el desarrollo: permiten cumplir los requerimientos de SGIP de forma profesional.
